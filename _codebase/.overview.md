# apps/ — Overview

All Newhaze application repositories managed as git submodules, plus shared infrastructure.

## App ecosystem

| Repo | URL | Stack | Role |
|---|---|---|---|
| `newhaze-api` | `api.newhaze.ar` | .NET 10 / ASP.NET Core | Central backend hub — sole layer touching Supabase |
| `newhaze-learn` | `learn.newhaze.ar` | React 18 + Vite | Learning platform — 15-topic curriculum with XP and badges |
| `newhaze-website` | `newhaze.ar` | Next.js 15 | Public marketing site — landing, blog, auth, products |
| `newhaze-inner-panel` | `deploy.newhaze.ar` | React 18 + Vite | Internal deploy dashboard (employee/developer only) |
| `newhaze-outer-panel` | TBD | Next.js | Public-facing user panel |
| `newhaze-consumer-panel` | TBD | Next.js | Consumer panel (early stage) |
| `newhaze-ui` | — | Vite + TypeScript | Shared design system and component library |

## Shared infrastructure

- `docker-compose.dev.yml` — local dev stack (run from `apps/` directory)
- `scripts/` — shared tooling scripts

## Auth model

All apps share a single SSO JWT issued by `newhaze-api`, validated via JWKS from Supabase Auth. No app calls Supabase directly. API types are auto-generated from `newhaze-api/docs/openapi/openapi.json` — never hand-write response shapes.

## Branch strategy

`dev` → `beta` → `main` for all repos. Never commit directly to `beta` or `main`.
