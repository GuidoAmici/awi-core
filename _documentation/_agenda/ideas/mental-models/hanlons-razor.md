# Hanlon's Razor

Never attribute to malice that which can be adequately explained by stupidity (or ignorance, incompetence, or carelessness).

A charitable interpretation heuristic. Most bad outcomes are not intentional.

## When to use
- When someone does something that hurts you or your team
- When a system fails in a way that feels targeted
- Before writing an angry message or escalating a conflict

## Example
A contractor delivers work late. Hanlon's Razor: they probably misjudged scope, had personal issues, or miscommunicated — not that they're sabotaging you.

An API returns unexpected data. Hanlon's Razor: the docs are wrong or the dev misunderstood the spec — not that someone changed it deliberately to break you.

## Pitfall
This is a prior, not a conclusion. If patterns repeat, update toward intentionality. Hanlon's Razor doesn't mean ignoring bad actors — it means not *starting* with that assumption.

Also: "stupidity" is shorthand. Real causes include: unclear communication, different context, burnout, missing information.
