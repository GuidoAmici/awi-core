# Inversion

Instead of asking "how do I succeed," ask "what would guarantee failure?" — then avoid those things.

Attributed to mathematician Carl Jacobi: *"Invert, always invert."* Popularized by Charlie Munger.

## When to use
- Pre-mortems: before starting a project, list everything that could kill it
- Unsticking yourself: if you can't find the right answer, find all the wrong ones
- Stress-testing plans: "what would have to be true for this to go badly?"

## Example
Want a successful product launch? Ask: what would make it a disaster?
- No one knows it exists → marketing problem
- People try it and churn → onboarding/value problem
- It breaks under load → infrastructure problem

Now solve for those. You've just done the same work, but more completely.

## Pitfall
Don't stop at identifying the negatives — you still have to build the positive path. Inversion is a filter, not a strategy.
