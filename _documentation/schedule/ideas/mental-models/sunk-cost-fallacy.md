# Sunk Cost Fallacy

Past costs — time, money, effort — that cannot be recovered should have zero influence on future decisions. They don't. We weight them heavily anyway.

The rational question is always: "Given where I am now, what's the best path forward?" The past is irrelevant.

## When to use (watch for it)
- Continuing a failing project because of how much was already invested
- Staying in a bad relationship/job/tool because of history
- Finishing a bad book because you already read 100 pages

## Example
"We've spent 6 months building this feature, we can't kill it now."
Rational reframe: the 6 months are gone regardless. The question is: does finishing this feature create more value than the next 2 months of alternatives?

"We've been using this library for years."
Rational reframe: does it still serve us better than alternatives, accounting only for migration cost vs future benefits?

## The tell
If your argument for continuing something relies heavily on what you've already invested, check for sunk cost reasoning.
