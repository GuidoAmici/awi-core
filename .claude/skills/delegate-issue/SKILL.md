# Delegate Issue

Dispatch GitHub issues labeled `ready-for-agent` to background agents.
`ready-for-agent` implies the issue was grilled and an employee was assigned in the agent brief.
Never fires without user confirmation.

## Step 1: Scan

Fetch `ready-for-agent` issues across all active org repos from `_data/users/current-user.json` → user-submodules.json.

For each active repo:
```bash
gh issue list --repo <owner/repo> \
  --label "ready-for-agent" \
  --json number,title,labels,comments \
  --state open
```

## Step 2: Validate each issue

For each issue, find the `## Agent Brief` comment. It must contain:
- `**Assigned employee:**` — a key present in `.claude/reference/employees.json`
- `**Model:**` — opus / sonnet / haiku

If either field is missing, skip the issue and warn:
```
⚠ #42 skipped — agent brief missing Assigned employee or Model. Re-run /triage to complete it.
```

## Step 3: Present dispatch list

```
N issues ready for dispatch:

  [1] Fix auth token expiry bug (#42, newhaze-api)
      → senior-developer · opus · $2.00

  [2] Write ADR for caching strategy (#17, my-awi-user)
      → technical-writer · sonnet · $1.00

Fire all? [a]  ·  Select [1,2…]  ·  Skip [s]
```

Wait for user input before proceeding.

## Step 4: Dispatch confirmed issues

For each confirmed issue:

1. **Load employee persona:**
   ```
   path = employees.json[assigned_employee]["path"]
   ```
   Read the full `.md` file at that path.

2. **Extract agent brief** from the issue comment (the `## Agent Brief` block).

3. **Build prompt:**
   ```
   <employee .md content>

   ---

   ## Your task

   <agent brief content, verbatim>

   When complete, save a summary to <AWI_ROOT>/info/outputs/YYYY-MM-DD-issue-<number>-<slug>.md
   Then comment on the issue: gh issue comment <number> --repo <owner/repo> --body "..."
   ```

4. **Fire delegate:**
   ```bash
   python3 .claude/skills/delegate/scripts/delegate_run.py \
     --prompt "<prompt>" \
     --model <model from brief> \
     --slug "issue-<number>-<short-title>" \
     --budget <budget matching model>
   ```

   Budget defaults: opus → 2.00 · sonnet → 1.00 · haiku → 0.20

5. **Comment on the issue:**
   ```bash
   gh issue comment <number> --repo <owner/repo> \
     --body "> *This was generated by AI.*
   
   🤖 Dispatched to background agent.
   **Slug:** \`issue-<number>-<short-title>\`
   **Employee:** \`<assigned_employee>\`
   **Started:** <datetime>"
   ```

## Step 5: Log

```bash
python3 .claude/skills/shared/scripts/log_command.py delegate-issue <outcome>
```

`<outcome>`: `completed` | `skipped` | `errored`

## Budget table

| Model  | Budget |
|--------|--------|
| opus   | 2.00   |
| sonnet | 1.00   |
| haiku  | 0.20   |

## Kill switch

Any running delegate can be stopped:
```bash
python3 .claude/skills/delegate/scripts/delegate_kill.py issue-<number>-<slug>
```

List all running delegates:
```bash
python3 .claude/skills/delegate/scripts/delegate_monitor.py
```
